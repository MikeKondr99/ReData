# Инфа для тестирования наборов

## Что нужно знать на старте
- Базовый OpenAPI: `http://localhost:5223/openapi/v1.json`.
- Главный тестовый коннектор: `00000000-0000-0000-0000-000000000000` (`Demo`).
- Основные эндпоинты:
  - `POST /api/transform` — прогон трансформаций без сохранения набора.
  - `POST /api/datasets` — создание набора.
  - `PUT /api/datasets/{id}` — обновление набора.
  - `GET /api/datasets/{id}` — проверка, что реально сохранилось.
  - `GET /api/datasets/{id}/export?fileType=...` — проверка исполнения набора на выгрузке.

## Практика по сериализации трансформаций
- Для полиморфных трансформаций безопасно передавать `"$type"` первым полем в объекте `transformation`.
- Если отправлять JSON руками, лучше использовать строковый литерал JSON для детерминированного порядка полей.
- Для сохраненных наборов обязательно проверять:
  - факт сохранения (`201/200`);
  - содержимое через `GET /api/datasets/{id}`;
  - исполнимость через `export`.

## Чеклист минимального прогона
- Создать набор с `select` + `where`.
- Проверить `GET /api/datasets/{id}` на фактическую структуру `transformations`.
- Проверить `export` для `fileType=0`.
- Для негативных сценариев:
  - невалидное выражение (`"(("`);
  - ручной JSON с перестановкой полей (`"$type"` не первым).

## Где чаще всего расходится ожидание
- OpenAPI и фактические ответы в ряде мест не полностью совпадают.
- В ответах ошибки иногда малоинформативны (`message = null`, пустой `errors`) или с утечкой stack trace.
