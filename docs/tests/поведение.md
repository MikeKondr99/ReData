# Поведенческие косяки API (не OpenAPI)

- Сохранение/обновление набора допускает невалидные трансформации (дубли alias, null-коллекции), которые затем падают на export с `500`.
- `PUT /api/datasets/{id}` не валидирует mismatch между path id и body id (молчаливое обновление path-объекта).
- Runtime-ошибки выражений (например, `1 / 0`) приводят к `500` с непоследовательным форматом ошибок.
- `POST /api/data-connectors` без `file` (multipart) может падать `500` вместо `400`.
- Для numeric out-of-range `fileType` в export возвращается `500` вместо валидационного `400`.
- `POST /api/transform` на `null`-структурах возвращает `400`, но с внутренними exception-текстами (`Непредвиденная ошибка ...`) вместо чистой валидации.
- Дубликат `name` в `POST /api/data-connectors` приводит к `500` (`DbUpdateException` по уникальному индексу) вместо управляемого конфликта.
- `POST /api/transform` принимает `pageSize = 0` и возвращает все строки (фактически без пагинации), вместо `400`.
- `POST /api/datasets` сохраняет пустые обязательные части трансформаций (`select.items=[]`, `orderBy.items=[]`, `where.condition=""`), после чего `export` падает `500`.
- `POST /api/transform` для некоторых невалидных структур возвращает `400`, но с пустой диагностикой (`message = null`, `errors = []`).
- Для одного и того же пайплайна preview (`POST /api/transform`) и `export` могут давать разную арифметику: `sum_amt / cnt` в preview считается как `int/int` (33), а в export как дробное (33.3333...).
- В preview возможна внутренняя несогласованность: `where` фильтрует по дробному значению, а в ответе поле отображается как `int` (например, видно `33`, но условие `> 33` проходит).
- Для `num` с большими и высокоточными значениями есть деградация точности (усечение значимых разрядов) уже на уровне данных/агрегаций.
- Очень малые `num` (проверено на `1e-400`) молча превращаются в `0` без валидационной ошибки.
- Сверхбольшие `num` (проверено на `1e+400`) переходят в `Infinity/-Infinity`: `transform` падает `500` на JSON-сериализации, а `export` отдает `Infinity` в CSV.
- Для степенных переполнений (`Pow`-каскад) `transform` возвращает `500`, но `export` может вернуть `200 OK` и затем оборвать stream (битый ответ).
