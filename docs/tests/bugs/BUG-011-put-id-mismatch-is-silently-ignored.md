# BUG-011: Несовпадение `id` в path и body при `PUT` молча игнорируется

## Статус
- Нужен фикс.

## Описание что не так
- `PUT /api/datasets/{id}` принимает `id` в body.
- Если `body.id` отличается от `{id}` в path, endpoint все равно возвращает `200` и обновляет запись из path.
- Ошибка в клиенте (неверный `body.id`) не обнаруживается и остается незаметной.

## Шаги воспроизведения
- Создать набор `A`.
- Вызвать `PUT /api/datasets/{A.id}` с `body.id = B.id` (другого набора).
- Получить `200`.
- Проверить:
  - обновился только набор `A`;
  - набор `B` не изменился.

## Ожидаемое поведение
- Если `body.id` присутствует и не совпадает с path `id`, должен возвращаться `400`.
- Либо `id` не должен приниматься в body вообще.

## Возвращенный ответ API
- `PUT`: `Status 200`.
- В ответе и фактическом состоянии изменяется набор из path.
- Набор, чей `id` передан в body, остается без изменений.
