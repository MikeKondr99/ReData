# BUG-020: `export` для отсутствующего набора возвращает `404`, но это не описано в OpenAPI

## Описание что не так
- `GET /api/datasets/{id}/export` для несуществующего `id` возвращает `404`.
- В OpenAPI для этого endpoint не описан `404` (указан только `204`).

## Шаги воспроизведения
- Вызвать `GET /api/datasets/{missing-id}/export?fileType=0`.
- Получить `404`.

## Ожидаемое поведение
- Либо добавить `404` в OpenAPI для endpoint export.
- Либо изменить фактическое поведение, если контракт должен быть другим.

## Возвращенный ответ API
- `Status 404`
- Body: пустой.
