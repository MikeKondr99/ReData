# BUG-004: Несоответствие OpenAPI и фактического статуса export

## Описание что не так
- В OpenAPI для `GET /api/datasets/{id}/export` указан ответ `204 No Content`.
- По факту при успешной выгрузке приходит `200` и тело файла (`text/csv`, attachment).
- Это ломает ожидания клиентов, которые опираются на спецификацию.

## Шаги воспроизведения
- Взять валидный набор.
- Выполнить `GET /api/datasets/{id}/export?fileType=0`.
- Получить `200` + файл вместо `204`.

## Ожидаемое поведение
- Спецификация и фактический ответ должны совпадать.

## Возвращенный ответ API
- Status `200`
- Headers: `Content-Type: text/csv`, `Content-Disposition: attachment; filename="...csv"`
- Ошибка не возвращалась; несоответствие именно в контракте OpenAPI (ожидался `204`).
