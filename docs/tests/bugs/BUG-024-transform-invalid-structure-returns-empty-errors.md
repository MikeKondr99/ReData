# BUG-024: `POST /api/transform` при невалидной структуре возвращает `400`, но без диагностик (`errors: []`)

## Описание что не так
- Для части невалидных трансформаций endpoint отвечает `400`, но тело фактически бесполезно для клиента:
  - `{"index":0,"message":null,"errors":[]}`
- Нет ни сообщения, ни привязки к полю, ни причины ошибки.
- Подтверждено на сценариях:
  - `select.items = []`
  - `orderBy.items = []`
  - `groupBy.groups = []` и/или `groupBy.items = []`

## Шаги воспроизведения
- Вызвать `POST /api/transform` с валидным `dataConnectorId`, `pageNumber`, `pageSize`.
- Передать одну из невалидных конфигураций выше.
- Получить `400` с пустым `errors`.

## Ожидаемое поведение
- Возвращать читаемую ошибку валидации с указанием проблемного поля и причины (минимум одно сообщение).

## Возвращенный ответ API
- `Status 400`
- Тело: `{"index":0,"message":null,"errors":[]}`
