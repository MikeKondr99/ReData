# BUG-019: Дубликат имени набора возвращает `400`, хотя в OpenAPI заявлен `409`

## Описание что не так
- Для `POST /api/datasets` в OpenAPI описан ответ `409` (конфликт).
- При попытке создать набор с существующим `name` API возвращает `400`.
- Это контрактный рассинхрон, ломающий клиентскую обработку конфликтов.

## Шаги воспроизведения
- Взять существующее имя набора (например, `Field and Vars`).
- Вызвать `POST /api/datasets` с этим именем.
- Получить `400`.

## Ожидаемое поведение
- Либо возвращать `409` в случае конфликта имени.
- Либо обновить OpenAPI и убрать/переписать `409`, если фактический контракт другой.

## Возвращенный ответ API
- `Status 400`
- Body: `Не выполнено указанное условие для 'name'.`
