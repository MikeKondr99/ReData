# BUG-002: Невалидное выражение сохраняется в набор, затем export падает с 500

## Описание что не так
- `POST /api/datasets` позволяет сохранить набор с невалидной трансформацией (`where` с `condition = "(("`), возвращает `201`.
- После этого `GET /api/datasets/{id}/export?fileType=0` возвращает `500`.
- Получается, в систему попадает набор, который гарантированно ломается на выполнении.

## Шаги воспроизведения
- Создать набор с валидной первой трансформацией-описанием и второй невалидной `where`.
- Убедиться, что создание вернуло `201`.
- Запросить `export` для этого набора и получить `500`.

## Ожидаемое поведение
- Невалидные выражения должны блокироваться при сохранении (или сохраняться как draft с явным статусом неисполняемого набора).
- На `export` не должно быть необработанного `500`.

## Возвращенный ответ API
- При сохранении набора: Status `201` (набор создан).
- При экспорте: Status `500`, Body: пустой.
