# BUG-023: Пустые обязательные части трансформаций сохраняются в набор, затем `export` падает `500`

## Описание что не так
- `POST /api/datasets` принимает некоторые явно невалидные/пустые трансформации и возвращает `201`.
- После сохранения `GET /api/datasets/{id}/export` для таких наборов возвращает `500`.
- Подтвержденные варианты:
  - `select.items = []` -> сохраняется, `export` = `500`.
  - `orderBy.items = []` -> сохраняется, `export` = `500`.
  - `where.condition = ""` -> сохраняется, `export` = `500`.

## Шаги воспроизведения
- Создать набор через `POST /api/datasets` с одним из вариантов выше.
- Убедиться, что ответ `201` и набор доступен через `GET /api/datasets/{id}`.
- Выполнить `GET /api/datasets/{id}/export?fileType=0`.
- Получить `500`.

## Ожидаемое поведение
- Невалидные пустые обязательные части должны отклоняться на этапе `POST/PUT` с `400` и явной ошибкой по полю.
- До `export` такие конфигурации не должны доходить.

## Возвращенный ответ API
- На создании: `Status 201`.
- На выгрузке: `Status 500` (для `select.items = []`, `orderBy.items = []`, `where.condition = ""`), тело пустое.
