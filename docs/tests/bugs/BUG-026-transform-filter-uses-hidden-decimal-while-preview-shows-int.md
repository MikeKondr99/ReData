# BUG-026: В `POST /api/transform` фильтр использует дробное значение, но в preview поле показано как `int`

## Описание что не так
- Для пайплайна с `ratio = sum_amt / cnt` preview отдает:
  - `fields.ratio.type = int`
  - значения `ratio` как целые (`33`, `113` и т.д.)
- Но `where ratio > X` работает так, как будто `ratio` дробное (`33.333...`, `113.333...`):
  - `ratio > 33` включает строку с отображаемым `ratio = 33`
  - `ratio > 33.5` уже исключает эту строку
- В итоге пользователь видит одни значения, а фильтрация выполняется по другим.

## Шаги воспроизведения
- Источник: `dataConnectorId = fa009d79-144f-460a-9399-2dddeb5e6769`.
- Пайплайн:
  - `groupBy team` с `cnt = COUNT()`, `sum_amt = SUM(amount)`
  - `select ratio = sum_amt / cnt`
  - `where ratio > 33`
- Выполнить `POST /api/transform`.
- Затем повторить с `where ratio > 33.5`.

## Ожидаемое поведение
- Либо preview и тип поля показывают фактическое значение, по которому идет фильтрация.
- Либо фильтрация выполняется по тем же приведенным значениям, которые возвращаются клиенту.
- В идеале одинаковая семантика и для preview, и для export.

## Возвращенный ответ API
- Для `where ratio > 33`: возвращаются `A`, `B`, `C`; у `B` отображается `ratio = 33`.
- Для `where ratio > 33.5`: возвращаются только `A`, `C`.
- `fields.ratio.type = int`.
