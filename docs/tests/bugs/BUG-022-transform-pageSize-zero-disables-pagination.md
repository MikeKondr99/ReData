# BUG-022: `POST /api/transform` принимает `pageSize = 0` и отключает пагинацию

## Описание что не так
- Endpoint валидирует `pageNumber >= 1` и `pageSize <= 100`, но не валидирует `pageSize >= 1`.
- При `pageSize = 0` запрос проходит (`200`) и возвращает все строки страницы, то есть фактически без лимита.
- Это ломает ожидаемую семантику пагинации и обходится без явного `400`.

## Шаги воспроизведения
- Вызвать `POST /api/transform` с валидным `dataConnectorId`, `pageNumber = 1`, `pageSize = 0`.
- Пример трансформации: `select` c `restOptions = Delete` и полем `id`.
- Сравнить с запросом `pageSize = 1..100`.

## Ожидаемое поведение
- Для `pageSize = 0` возвращать `400` с понятной ошибкой валидации (`pageSize` должен быть >= 1).

## Возвращенный ответ API
- `Status 200`.
- `total = 5`, `count(data) = 5` (для тестового источника на 5 строк), то есть фактически выдан весь набор.
